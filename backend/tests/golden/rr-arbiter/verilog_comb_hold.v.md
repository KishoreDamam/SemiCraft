# Round-Robin Arbiter

An 8-way round-robin arbiter: it grants a shared resource to at most one requester per cycle (one-hot grant, asserted the same cycle as the request), rotating priority so that under sustained contention every requester is served at least once every 8 cycles.

## Ports

### Clocking

Clock and reset for the priority pointer (and grant register).

| Port | Direction | Description |
| --- | --- | --- |
| `clk` | input | Clock; the priority pointer (and grant register) update on the rising edge. |
| `rst_n` | input | Sync reset, active-low reset input. |

### Arbitration

Per-requester request lines, the one-hot grant, and its valid flag (|grant).

| Port | Direction | Description |
| --- | --- | --- |
| `req` | input | 8 request lines; req[i] asserts that requester i wants the resource. |
| `grant` | output | One-hot grant (all-zero when no request is pending), combinational (asserted the same cycle as the request). |
| `grant_valid` | output | High whenever a grant is asserted; equal to |grant. |

## Configuration

- Requesters: 8 (req/grant are 8 bits, pointer is 3 bits)
- Grant style: combinational (grant asserts the same cycle, no latency, longer critical path from req through the priority network)
- Arbitration policy: hold — while the granted requester keeps asserting req it keeps the grant; priority rotates only once it deasserts
- Reset: sync, active-low
- Output language: Verilog-2001

## Assumptions

- A single free-running clock drives the arbiter; req is sampled synchronously to it.
- Requesters keep req asserted until granted (there is no separate grant-acknowledge handshake).

## Limitations

- Unweighted round-robin: every requester has equal priority in the rotation; there is no way to bias or prioritise a requester.
- No grant/acknowledge protocol: grant is asserted per the fixed latency of the chosen grant_style; the arbiter does not wait for an ack before rotating.
- N (8) is fixed at generation time: the priority network is unrolled for this exact requester count and is not a runtime parameter.
- Combinational grant places the full two-pass priority network on the req-to-grant critical path; at large N consider the registered style for timing closure.
