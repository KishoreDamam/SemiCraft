# Clock Divider

A clock divider by 1024: a free-running counter produces clk_out as a pulse (single-cycle clock-enable tick).

_Generated by SemiCraft 0.1.0 — config hash `a1a9207f5113`._

## Ports

### Clocking

Input clock and reset for the divide counter.

| Port | Direction | Description |
| --- | --- | --- |
| `clk` | input | Input clock; the divide counter runs on its rising edge. |
| `rst` | input | Async reset, active-high reset input. |

### Data

Single-cycle enable pulse, asserted once every DIV=1024 input cycles

| Port | Direction | Description |
| --- | --- | --- |
| `clk_out` | output | Single-cycle enable pulse, asserted once every DIV=1024 input cycles. |

## Configuration

- Divide ratio: 1024 (param DIV)
- Output style: pulse (single-cycle clock-enable tick)
- Counter width: 10 bits (clog2(DIV))
- Reset: async, active-high
- Output language: SystemVerilog

## Assumptions

- A single free-running input clock drives the divider.
- The consumer of clk_out either treats it purely as a data signal (toggle style) or as a synchronous clock-enable on the original clk (pulse style), not as a low-skew clock network.

## Limitations

- The generated clk_out is a data signal produced by an ordinary flip-flop — it is NOT a low-skew clock and must not be routed onto a clock network or used to directly clock other flip-flops in a real FPGA/ASIC design. Use a PLL/MMCM for a genuine divided clock, or prefer the 'pulse' clock-enable style and keep all logic on the original clk (see the clean-rtl reference).
- No fractional division: divide_by must be an even integer, so only even divisors are representable with a 50%-duty toggle output.
