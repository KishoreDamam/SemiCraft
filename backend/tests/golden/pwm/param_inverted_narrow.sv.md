# PWM Generator

A 6-bit PWM generator: a free-running counter compared against a fixed parameter (DUTY) produces pwm_out, inverted, with a 64-cycle period.

_Generated by SemiCraft 0.1.0 — config hash `349000b7556e`._

## Ports

### Clocking

Clock and reset for the free-running period counter.

| Port | Direction | Description |
| --- | --- | --- |
| `clk` | input | Clock; the period counter increments on the rising edge. |
| `rst_n` | input | Sync reset, active-low reset input. |

### Data

Generated PWM output (duty cycle is a fixed parameter).

| Port | Direction | Description |
| --- | --- | --- |
| `pwm_out` | output | PWM output, inverted (active-low); combinational comparison of cnt against duty (64-cycle period). |

## Configuration

- Resolution: 6 bits (period = 2^6 = 64 cycles)
- Duty-cycle source: a fixed parameter (DUTY)
- Output polarity: inverted (active-low)
- Reset: sync, active-low
- Output language: SystemVerilog

## Assumptions

- A single free-running clock drives the period counter.
- duty (when a runtime port) is stable while sampled by the combinational comparison; the caller is responsible for updating it synchronously if it varies.

## Limitations

- duty=0 makes the (uninverted) output constant low for the whole period (constant high when invert_output is set), since cnt is never less than 0.
- duty=63 (the maximum representable value) yields the output high for 63 of the 64 counter values — one cycle short of 100% duty, since the comparison is strict (cnt < duty), not cnt <= duty.
- No dead-time, phase-shift, or center-aligned modes; this is a single-channel, edge-aligned PWM only.
